# 8. 下載格式

支援回傳檔名或直接二進位串流。

---

## 回傳格式

### Base64
**適合小型檔案下載**

**優點：**
- 可以回傳其他資訊（如成功訊息）

**缺點：**
- 檔案變大 33%
- 記憶體負擔大

**回傳範例格式：**
```json
{
  "statusCode": "200",
  "messageCode": null,
  "message": "執行成功",
  "data": {
    "exportFileData": "77u/LCwsLCzlpKfkuovntIDs...",
    "exportFileName": "大書紀資料明細資料表.csv"
  }
}
```

---

### application/octet-stream
**適合大型檔案下載（需要額外處理 response header Content-Disposition 來取得檔名）**

**優點：**
- 效能最佳
- 支援大檔案
- 下載體驗好

**缺點：**
- 無法回傳其他訊息
